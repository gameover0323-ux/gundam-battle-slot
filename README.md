# GUNDAM BATTLE SLOT

## 概要
ガンダム系ボードゲーム開発用リポジトリ

## ファイル構成
- index.html : 画面本体
- js/main.js : 入口
- js/battle_system.js : 共通戦闘処理
- js/ui.js : UI処理
- js/units/ : 機体ごとの個別データと処理

## 現在の方針
- システム側は共通フック中心
- 機体追加ファイルで完結できる範囲を増やす

## 厳守ルール
- 推測で仕様を書かない
- 説明文ではなく内部データ優先
- 回避、必中、軽減不可の処理は既存仕様を厳守

## 今後の実装方針

## セレクト画面について
- 最終仕様としては機体一覧を作品別、歴代順にソートできるように並べたい。

## バトルシステムについて
最終的にはHPを0にした時、カウンター勝利案件がなければそのまま勝敗ポップアップが出現し、タイトルにもどる。


## 回避について
- 必中属性以外の攻撃を無効化できる。
何らかの特殊行動リソースとして使われることが大半である。イメージはスピードや機動力。
- 体ごとの回避ストック上限まで持つことが出来、上限を超えた数値はターン終了後に端数切り捨てされる。
- ただし、回避ストックが上昇する形態から変化した場合は原則最大ストック分保持され、UIが赤になる。これを赤上限状態と呼称する。

## 属性について
- 内部データ的に今後ギミックとして使われると思われる属性。
- 格闘→格闘カウンターなどの判定のために必要
- 射撃→格闘と同様。
- サイコミュ→現在は該当所持する機体がいない。ファンネルだけでなく、ドラグーンやGNファングもサイコミュ属性に該当する。
- ビーム→対ビーム性能を持つ武装などに対応させる。
-  必中→特殊行動や特性で定義されない限りは原則回避不可能。
- 軽減不可→特殊行動や特性で定義されない限り原則数値の減衰は不可能。
- 回復→稀に複合的に組み込まれているスロットもある。
- 回避→稀に複合的に組み込まれているスロットもある。
- 強化→換装や時限強化を行う。これは実際には弱体化の場合も強化属性として定義する。

 
- 特殊行動やスロット行動で新たに付与される属性の場合、属性が黄色で追加される。

- これらの判定は説明文依存禁止。説明文はユーザーが読んで理解するためのものなので、コードでのソートを組み込むことは原則避けること。

## 攻撃段階について
- ヒット数に応じてQTEが発生する。このタイミングで使える特殊行動が反応系。

## 特殊行動分類について
- 各種系統の名前は構造上の通称。
- 自動系→プレイヤーが実行ボタンを押すことなく勝手に発動するもの。特性が主にそれに当たる。
- 行動系→プレイヤーの行動権がある際に常に実行可能になるもの。条件が揃った時に発動可能なものも。変形が主にそれに当たる。
- 反応系→相手プレイヤーの行動に反応して行うもの。シールドが主にそれに当たる。

## 2on2仕様について
- 2on2においてはHP各種の扱いが異なる。
  # 統合型→2機のHPが合算される。
  - 回避ストック最大値や所持数が合算される。
  - HP減算は内部的にはお互いの数値比率計算の元、分散される。
  - 一機体扱いになるので、減算処理や回避は二機体分を一ステータスで使える。シールドを二機体分で半減の半減、見切り行動のターン数増加も単純に二機体ぶんのスロット行動が回せる。
  - 各期待の特性も一機体として反映。常時ダメージ-10等の処理、ゼロシステム等の特殊な回避も適用。
  - 統合型比率分散の考え→統合型編成前の元数値が100:50のとき、被ダメージ比率は2:1。100ダメージを受けた場合は67＋33ダメージになる。
  - 統合型→分散型切り替えの以降計算→元HPを関数保存し、さらにそれぞれA1回復値、A2回復値、A1回避値、A2回避値を設定する。元HPの比率を保存し、統合型総ダメージ値を蓄積していく。分散時HPは（A1元HP値＋A1回復値）－(全体ダメージ÷A1HP比率)のように分配する。回避数値は元比率を参考に、最大値になった場合を除いて比率通りに分配する。
# 分散型→HPが独立する上、1機をフォーカスし、QTE全てを1機で受ける。
- 相方は援護防御として、回避を1消費して1発の攻撃ダメージを半減して受けることができる。

- スロット行動については同時にバラバラの乱数で抽選し、戦闘ログに反映する。

## 特殊仕様:挑発
- 5ターンのクールタイム、5ターンの効果時間のある特殊ルール。
- 発動者は分散型に5ターン間固定される。その際に相手プレイヤーの1機を指定する。

- 相手プレイヤーは指定された1機を分散型にしてフォーカスした時、相手プレイヤーからデメリット効果を受けることはない。
- 指定されていない機体で分散型、もしくは統合型にした場合、被ダメージが1.5倍になってしまう。

## 決戦システム
- 挑発に挑発で応じる時、お互いのフォーカス機体への与ダメージが2倍になる。

## 打破賭け(ダメージベットレース)
- 決戦を自発的に解除する方法。
- お互いに10ターンまでベットすることが出来、シミュレーションして10ターンまでの間でダメージ数の合計が多い方がボーナスターンを得ることができる。
- ボーナスターン(行動権)の付与は以下の通り
- 10vs10=5ターン
- 10vs9:8=4ターン
- 10vs7:6=3ターン
- 10vs5:4=2ターン
- 10vs3以下=1ターン

- 9:8vs9:8=4ターン 
- 9:8vs7:6=3ターン
- 9:8vs5:4=2ターン
- 9:8vs3以下=1ターン

- 7:6vs7:6=3ターン
- 7:6vs5:4=2ターン
- 7:6vs3以下=1ターン

- 5:4vs5:4=2ターン
- 5:4vs3以下=1ターン

- 3以下vs3以下=1ターン

- 3以下は0も含む。

- 勝者は上記の行動権ボーナスを得ることができる。
- シミュレーションは実際の強化ターン数も消費してしまうが、逆にシミュレーション中の強化状態は引き継ぐ。
